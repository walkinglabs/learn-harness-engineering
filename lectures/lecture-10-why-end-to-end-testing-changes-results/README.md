# Lecture 10. Why Structure and Taste Must Be Enforced

## Guiding Question

Why do architecture and engineering taste have to be enforced mechanically in
agent-first systems?

## Why This Matters

Agent-generated systems drift quickly if structure and taste exist only as human
preferences. Harnesses preserve quality by turning them into enforceable
constraints.

## Core Ideas

- Architecture rules drift without enforcement.
- Structural tests and lints make boundaries enforceable.
- Taste invariants convert review feedback into reusable harness logic.
- Strict boundaries make future agent work easier and safer.
