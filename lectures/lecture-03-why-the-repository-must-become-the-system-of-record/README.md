# Lecture 03. Why the Repository Must Become the System of Record

## Guiding Question

Why must knowledge live inside the repository rather than outside it?

## Why This Matters

Agents can only reliably use knowledge they can access while working. If key
decisions live in chat threads, docs apps, or human memory, the harness cannot
count on them.

## Core Ideas

- Agent-invisible knowledge is operationally nonexistent.
- The repository becomes more than code: it becomes the system of record.
- Specs, plans, references, and operational notes should live close to the work.
